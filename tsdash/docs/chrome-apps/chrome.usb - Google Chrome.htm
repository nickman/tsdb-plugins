Description: Use the chrome.usb API to interact with connected USB devices. This API provides access to USB operations from within the context of an app. Using this API, apps can function as drivers for hardware devices.
Availability: Stable since Chrome 26.
Permissions: "usb"
Learn More: Accessing Hardware Devices

chrome.usb reference

Types

Direction

enum of "in", or "out"
Direction, Recipient, RequestType, and TransferType all map to their namesakes within the USB specification.

Device

properties of Device

device ( integer )

The id of the USB device. It remains unchanged until the device is unplugged.

vendorId ( integer )

productId ( integer )

ConnectionHandle

properties of ConnectionHandle

handle ( integer )

The id of the USB connection handle.

vendorId ( integer )

productId ( integer )

GenericTransferInfo

properties of GenericTransferInfo

direction ( Direction )

The direction of this transfer.

endpoint ( integer )

length ( optional integer )

If this is an input transfer then this field indicates the size of the input buffer. If this is an output transfer then this field is ignored.

data ( optional arraybuffer )

If this is an output transfer then this field must be populated. Otherwise, it will be ignored.

TransferResultInfo

properties of TransferResultInfo

resultCode ( optional integer )

A value of 0 indicates that the transfer was a success. Other values indicate failure.

data ( optional arraybuffer )

If the transfer was an input transfer then this field will contain all of the input data requested.

findDevices

chrome.usb.findDevices(object options, function callback)

Finds USB devices specified by the vendorId/productId/interfaceId tuple and, if permissions allow, opens them for use.

On Chrome OS, you can specify the interfaceId. In that case the method will request access from permission broker in the same way as in |requestUsbAcess|.

If the access request is rejected, or the device is failed to be opened, its connection handle will not be created or returned.

Calling this method is equivalent to calling |getDevices| followed by a series of |requestAccess| (if it is on ChromeOs) and |openDevice| calls, and returning all the successfully opened connection handles.

Parameters

options ( object )

The properties to search for on target devices.

Properties

vendorId ( integer )

productId ( integer )

interfaceId ( optional integer )

The interface id to request access against. Only available on ChromeOS. It has no effect on other platforms.

callback ( function )

Invoked with the opened ConnectionHandle on complete.

The callback parameter should specify a function that looks like this: 

function(array of ConnectionHandle handles) {...};

handles ( array of ConnectionHandle )

closeDevice

chrome.usb.closeDevice(ConnectionHandle handle, function callback)

Closes a connection handle. Invoking operations on a device after it has been closed is a safe operation, but causes no action to be taken.

Parameters

handle ( ConnectionHandle )

The connection handle to close.

callback ( optional function )

The callback to invoke once the device is closed.

If you specify the callback parameter, it should specify a function that looks like this: 

function() {...};

Lists all the interfaces on the USB device.

Parameters

handle ( ConnectionHandle )

The device from which the interfaces should be listed.

callback ( function )

The callback to invoke when the interfaces are enumerated.

The callback parameter should specify a function that looks like this: 

function(array of object descriptors) {...};

descriptors ( array of object )

Properties of each object

interfaceNumber ( integer )

alternateSetting ( integer )

interfaceClass ( integer )

interfaceSubclass ( integer )

interfaceProtocol ( integer )

description ( optional string )

endpoints ( array of object )

Properties of each object

address ( integer )

type ( enum of "control", "interrupt", "isochronous", or "bulk" )

direction ( Direction )

maximumPacketSize ( integer )

synchronization ( optional enum of "asynchronous", "adaptive", or "synchronous" )

Used for isochronous mode.

usage ( optional enum of "data", "feedback", or "explicitFeedback" )

pollingInterval ( optional integer )

If this is an interrupt endpoint, this will be 1-255.

claimInterface

chrome.usb.claimInterface(ConnectionHandle handle, integer interfaceNumber, function callback)

Claims an interface on the specified USB device. Before you can transfer data with endpoints, you must claim their parent interfaces. Only one connection handle on the same host can claim each interface. If the interface is already claimed, this call will fail.

You shall call releaseInterface when the interface is not needed anymore.

Parameters

handle ( ConnectionHandle )

The device on which the interface is to be claimed.

interfaceNumber ( integer )

callback ( function )

The callback to invoke once the interface is claimed.

The callback parameter should specify a function that looks like this: 

function() {...};

callback ( function )

controlTransfer

chrome.usb.controlTransfer(ConnectionHandle handle, object transferInfo, function callback)

Performs a control transfer on the specified device. See the ControlTransferInfo structure for the parameters required to make a transfer.

Conceptually control transfer talks to the device itself. You do not need to claim interface 0 to perform a control transfer.

Parameters

handle ( ConnectionHandle )

A connection handle to make the transfer on.

transferInfo ( object )

The parameters to the transfer. See ControlTransferInfo.

Properties

direction ( Direction )

The direction of this transfer.

recipient ( enum of "device", "interface", "endpoint", or "other" )

The intended recipient for this transfer.

requestType ( enum of "standard", "class", "vendor", or "reserved" )

The type of this request.

request ( integer )

value ( integer )

index ( integer )

length ( optional integer )

If this transfer is an input transfer, then this field must be set to indicate the expected data length. If this is an output transfer, then this field is ignored.

data ( optional arraybuffer )

The data payload carried by this transfer. If this is an output transfer then this field must be set.

callback ( function )

Invoked once the transfer has completed.

The callback parameter should specify a function that looks like this: 

function(TransferResultInfo info) {...};

info ( TransferResultInfo )

isochronousTransfer

chrome.usb.isochronousTransfer(ConnectionHandle handle, object transferInfo, function callback)

Performs an isochronous transfer on the specific device.

Parameters

handle ( ConnectionHandle )

A connection handle to make the transfer on.

transferInfo ( object )

The parameters to the transfer. See IsochronousTransferInfo.

Properties

transferInfo ( GenericTransferInfo )

All of the normal transfer parameters are encapsulated in the transferInfo parameters. Note that the data specified in this parameter block is split along packetLength boundaries to form the individual packets of the transfer.

packets ( integer )

The total number of packets in this transfer.

packetLength ( integer )

The length of each of the packets in this transfer.

callback ( function )

Invoked once the transfer has been completed.

The callback parameter should specify a function that looks like this: 

function(TransferResultInfo info) {...};

info ( TransferResultInfo )

resetDevice

chrome.usb.resetDevice(ConnectionHandle handle, function callback)

Tries to reset the USB device and restores it to the previous status. If the reset fails, the given connection handle will be closed and the USB device will appear to be disconnected then reconnected. In that case you must call |getDevices| or |findDevices| again to acquire the device.

Parameters

handle ( ConnectionHandle )

A connection handle to reset.

callback ( function )

Invoked once the device is reset with a boolean indicating whether the reset is completed successfully.

The callback parameter should specify a function that looks like this: 

function(boolean result) {...};

result ( boolean )
