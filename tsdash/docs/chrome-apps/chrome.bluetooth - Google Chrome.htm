chrome.bluetooth reference

Types

AdapterState

properties of AdapterState

address ( string )

The address of the adapter, in the format 'XX:XX:XX:XX:XX:XX'.

name ( string )

The human-readable name of the adapter.

powered ( boolean )

Indicates whether or not the adapter has power.

available ( boolean )

Indicates whether or not the adapter is available (i.e. enabled).

discovering ( boolean )

Indicates whether or not the adapter is currently discovering.

Device

properties of Device

address ( string )

The address of the device, in the format 'XX:XX:XX:XX:XX:XX'.

name ( optional string )

The human-readable name of the device.

paired ( optional boolean )

Indicates whether or not the device is paired with the system.

connected ( optional boolean )

Indicates whether the device is currently connected to the system.

Profile

properties of Profile

uuid ( string )

Unique profile identifier, e.g. 00001401-0000-1000-8000-00805F9B23FB

name ( optional string )

Human-readable name of the Profile, e.g. "Health Device"

channel ( optional integer )

The RFCOMM channel id, used when the profile is to be exported to remote devices.

psm ( optional integer )

The LS2CAP PSM number, used when the profile is to be exported to remote deviecs.

requireAuthentication ( optional boolean )

Specifies whether pairing (and encryption) is required to be able to connect.

requireAuthorization ( optional boolean )

Specifies whether user authorization is required to be able to connect.

autoConnect ( optional boolean )

Specifies whether this profile will be automatically connected if any other profile of device also exporting this profile connects to the host.

version ( optional integer )

Specifies the implemented version of the profile.

features ( optional integer )

Specifies the profile-specific bit field of features the implementation supports.

Socket

properties of Socket

device ( Device )

The remote Bluetooth device associated with this socket.

profile ( Profile )

The remote Bluetooth profile associated with this socket.

id ( integer )

An identifier for this socket that should be used with the read/write/disconnect methods.

OutOfBandPairingData

properties of OutOfBandPairingData

hash ( arraybuffer )

Simple Pairing Hash C. Always 16 octets long.

randomizer ( arraybuffer )

Simple Pairing Randomizer R. Always 16 octets long.

Methods

addProfile

chrome.bluetooth.addProfile(Profile profile, function callback)

Registers the JavaScript application as an implementation for the given Profile; if a channel or PSM is specified, the profile will be exported in the host's SDP and GATT tables and advertised to other devices.

Parameters

profile ( Profile )

callback ( function )

The callback parameter should specify a function that looks like this: 

function() {...};

removeProfile

chrome.bluetooth.removeProfile(Profile profile, function callback)

Unregisters the JavaScript application as an implementation for the given Profile; only the uuid field of the Profile object is used.

Parameters

profile ( Profile )

callback ( function )

The callback parameter should specify a function that looks like this: 

function() {...};

getAdapterState

chrome.bluetooth.getAdapterState(function callback)

Get information about the Bluetooth adapter.

Parameters

callback ( function )

Called with an AdapterState object describing the adapter state.

The callback parameter should specify a function that looks like this: 

function(AdapterState result) {...};

result ( AdapterState )

getDevices

chrome.bluetooth.getDevices(object options, function callback)

Get a bluetooth devices known to the system. Known devices are either currently paired, or have been paired in the past.

Parameters

options ( object )

Controls which devices are returned and provides |deviceCallback|, which is called for each matching device.

Properties

profile ( optional Profile )

Only devices providing |profile| will be returned.

deviceCallback ( function )

Called for each matching device. Note that a service discovery request must be made to each non-matching device before it can be definitively excluded. This can take some time.

Parameters

device ( Device )

callback ( function )

Called when the search is completed. |options.deviceCallback| will not be called after |callback| has been called.

The callback parameter should specify a function that looks like this: 

function() {...};

getProfiles

chrome.bluetooth.getProfiles(object options, function callback)

Returns the set of exported profiles for the device specified in options. This function will not initiate a connection to the remote device.

Parameters

options ( object )

Properties

device ( Device )

The remote Bluetooth device to retrieve the exported profiles list from.

callback ( function )

The callback parameter should specify a function that looks like this: 

function(array of Profile result) {...};

result ( array of Profile )

getServices

chrome.bluetooth.getServices(object options, function callback)

Get a list of services provided by a device.

Parameters

options ( object )

Properties

deviceAddress ( string )

The address of the device to inquire about. |deviceAddress| should be in the format 'XX:XX:XX:XX:XX:XX'.

callback ( function )

The callback parameter should specify a function that looks like this: 

function(array of object result) {...};

result ( array of object )

Properties of each object

name ( string )

The name of the service.

uuid ( optional string )

The UUID of the service.

setOutOfBandPairingData

chrome.bluetooth.setOutOfBandPairingData(object options, function callback)

Set the Out of Band Pairing data for a remote device. Any previous Out Of Band Pairing Data for this device is overwritten.

Parameters

options ( object )

The options for this function.

Properties

address ( string )

The address of the remote device that the data should be associated with. |deviceAddress| should be in the format 'XX:XX:XX:XX:XX:XX'.

data ( optional OutOfBandPairingData )

The Out Of Band Pairing Data. If this is omitted, the data for the device is cleared instead.

callback ( optional function )

Called to indicate success or failure.

If you specify the callback parameter, it should specify a function that looks like this: 

function() {...};

startDiscovery

chrome.bluetooth.startDiscovery(object options, function callback)

Start discovery. Discovered devices will be returned via the |onDeviceDiscovered| callback. Discovery will fail to start if it is already in progress. Discovery can be resource intensive: stopDiscovery should be called as soon as possible.

Parameters

options ( object )

The options for this function.

Properties

deviceCallback ( function )

Called for each device that is discovered.

Parameters

device ( Device )

callback ( optional function )

Called to indicate success or failure.

If you specify the callback parameter, it should specify a function that looks like this: 

function() {...};
